# Audit Log Validation Report

This report evaluates the logging system against the requirement to log every state-changing event with a timestamp, user hash, role, action, and before/after states (where applicable), supporting the cross-institutional trust model (Section 1.1) and proctoring/academic integrity standards.

---

## 1. Audit Log Database Structure Analysis

The current database structure utilizes the `AuditLog` model mapped via Prisma in `backend/prisma/schema.prisma` (lines 487-498):

```prisma
model AuditLog {
  id        String      @id @default(uuid())
  userId    String?
  action    AuditAction
  details   String?     // Description or JSON metadata
  ipAddress String?
  createdAt DateTime    @default(now())

  user User? @relation(fields: [userId], references: [id], onDelete: SetNull)

  @@map("audit_logs")
}
```

And the `AuditAction` enum (lines 478-486):
```prisma
enum AuditAction {
  LOGIN
  PROGRAM_CREATED
  PROGRAM_UPDATED
  ASSESSMENT_PUBLISHED
  CREDENTIAL_ISSUED
  ROLE_CHANGED
}
```

## 3. Database Schema Integrity & Structural Gaps

> [!CAUTION]
> Beyond missing enum values, two high-severity architectural bugs are present in the `AuditLog` model design that violate privacy and structural integrity guidelines:

### 3.1 Missing Historical Role Column
* **Defect**: The model lacks a `role` column. Joining dynamically to the live `User` model on `userId` to retrieve roles will fetch the user's *current* role. If a Student becomes a Faculty member later, historical audit trails will incorrectly state that a Faculty member triggered Student-only events.
* **Remediation**: Add a static `userRole` (e.g., `UserRole` enum) column directly to the `AuditLog` table to preserve the user's role *at the exact moment the event occurred*.

### 3.2 Exposed PII / Lack of User Hash Anonymization
* **Defect**: The blueprint specifies storing a "user hash" for audits. Currently, `AuditLog` links directly to raw `userId` (a UUID foreign key) and retrieves cleartext `User` details. This exposes PII globally in audit extracts and violates the zero-knowledge institutional model.
* **Remediation**: Add a `userHash` field (a SHA-256 hash generated by combining `userId` and an environment-level salt) to identify actors anonymously in audit trails, separating audit logs from direct identity rows unless explicitly de-anonymized by authorized admin keys.

## 4. Log Immutability & Multi-Tenancy Querying

### 4.1 Immutability
- **Current State**: The `AuditLog` table has no database constraints preventing updates or deletions.
- **DevOps Recommendation**: Enforce write/append-only permissions at the database level by revoking `UPDATE` and `DELETE` access on the `audit_logs` table for application database connections.

### 4.2 Multi-Tenant Querying (Section 1.1)
- **Current State**: Logs are queried globally using `AuditService.getLogs()`.
- **Gap**: There is no direct query filtering by University / Institution scope.
- **Backend Recommendation**: Update `getLogs` filters to query by institution ID linked to the user record, ensuring Partner-University Admins can only query audit logs belonging to their own students and faculty:
  ```typescript
  where: {
    user: {
      institutionId: requestingAdmin.institutionId
    }
  }
  ```

## 5. Suggested Prisma Schema Update

To implement the audit requirements fully, update `backend/prisma/schema.prisma` as follows:

```prisma
model AuditLog {
  id          String      @id @default(uuid())
  userId      String?
  userHash    String      // Hashed/anonymized identifier (SHA-256 of userId + salt)
  userRole    UserRole    // Captures role at time of action
  action      AuditAction
  details     String?     // JSON metadata of before/after state
  ipAddress   String?
  createdAt   DateTime    @default(now())

  user User? @relation(fields: [userId], references: [id], onDelete: SetNull)

  @@map("audit_logs")
}

enum AuditAction {
  LOGIN
  ROLE_CHANGED
  PROGRAM_CREATED
  PROGRAM_UPDATED
  ASSESSMENT_PUBLISHED
  CREDENTIAL_ISSUED
  
  PROCTORING_VIOLATION

  ACCESS_GRANTED
  LEADERBOARD_FROZEN
```
